## 1. Что такое husky-хук
- [ ] Git-хуки в принципе: скрипты, которые git запускает на события (commit, push и т.д.).
- [ ] Что делает husky: менеджер git-хуков; хранит их в `.husky/` (в репо, версионируются)
      и активирует автоматически на `pnpm install` через `prepare: husky`.
- [ ] Зачем в MN: `.husky/pre-push` блокирует прямой push в `main` (только через PR).
      Это защита от ошибок; обходится `--no-verify`.
- [ ] Как проверить: `git config core.hooksPath` → `.husky/_`.

## 2. Как сделать песочницу (изоляция для агента)
- [ ] Проблема: агент в основном терминале видит `~/.ssh`, `~/.config/gh`, `.env` —
      урезанный токен не спасает, если окружение общее.
- [ ] Варианты по возрастанию изоляции: отдельный OS-пользователь → Docker / dev container → VM.
      Плюсы/минусы, что проще на macOS.
- [ ] Практика: минимальный `.devcontainer/` или Dockerfile — репо примонтирован,
      в env только нужный `GH_TOKEN`.
- Теория и настройка токена уже расписаны → `docs/agent-security.md`.
  Здесь остаётся практика: собрать рабочий `.devcontainer/`.

## 3. Ликбез: corepack и pnpm
- [ ] pnpm: менеджер пакетов (альтернатива npm/yarn); зачем в монорепе (workspaces,
      экономия места, скорость).
- [ ] corepack: встроен в Node, ставит/фиксирует версию pnpm по полю `packageManager`
      в package.json. Команда `corepack enable`.
- [ ] Связка в MN: `packageManager: pnpm@9.x` + `corepack enable` перед `pnpm install`.
- [ ] Базовые команды pnpm: `install`, `add`, `--filter`, workspaces.
