## Контекст прошлой сессии (2026-07-21)

Закрыт **P01-T1**: fine-grained токен `mn-agent` создан и проверен — `Administration`
закрыт полностью, 403 отдаёт не только `DELETE .../branches/main/protection`, но и
`GET`. Токен не может ни прочитать, ни изменить защиту `main`.

Пункт «прописать `GH_TOKEN` в окружение» снят сознательно (см. `decisions-log`):
функционально то же, что текущий `gh auth`, а изоляцию `~/.config/gh` даёт только
песочница. Поэтому в «Готово, когда» пункт про `~/.ssh` и `~/.config/gh` остался
открытым и честно ждёт P01-T2.

**Операционный урок.** PR #10 смёржен squash'ем — в `main` уехал один коммит с
новым SHA, а рабочая ветка продолжала нести оригинал, что дало конфликт на ровном
месте. При squash-мёрже ветку, которая живёт дольше одного PR, надо ребейзить на
свежий `main` сразу после мёржа предыдущего. Ещё: `gh pr list` по умолчанию
показывает только открытые PR — для проверки «был ли уже PR» нужен `--state all`.
