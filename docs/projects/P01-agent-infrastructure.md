# P01 — Инфраструктура для работы с агентом

**Статус:** 🚧 в работе
**Цель:** G2 — безопасная и воспроизводимая среда разработки
**Обновлён:** 2026-07-21

## Зачем

Агент должен работать с урезанными правами в изолированном окружении, а не с
полным доступом к рабочей машине. Теория и настройка расписаны в
`docs/agent-security.md` — здесь остаётся довести до реально работающего состояния.

## Готово, когда

- [x] Агент запускается с токеном `mn-agent`, а не с личным `gh auth login`
- [ ] У агента нет доступа к `~/.ssh` и `~/.config/gh`
- [ ] Любой клон репы поднимается одной командой

## Задачи

### P01-T1 — Урезанный токен ✅
Пошаговая инструкция готова в `docs/agent-security.md`.

- [x] Создать fine-grained токен `mn-agent` (Administration = No access) — 2026-07-21
- [x] Проверить урезанность: `DELETE .../branches/main/protection` → 403 — 2026-07-21

Проверка 2026-07-21: `Administration` закрыт полностью — 403 даёт не только
`DELETE`, но и `GET` protection. Токен не может ни прочитать, ни изменить защиту `main`.

Токен лежит в `~/.config/gh/hosts.yml` и используется через `gh auth`. Отдельный
перевод на переменную `GH_TOKEN` решили не делать — 2026-07-21: функционально
разницы нет, а изоляцию `~/.config/gh` всё равно даёт только песочница (P01-T2).
