# P01 — Инфраструктура для работы с агентом

**Статус:** 🚧 в работе
**Цель:** G2 — безопасная и воспроизводимая среда разработки
**Обновлён:** 2026-07-20

## Зачем

Агент должен работать с урезанными правами в изолированном окружении, а не с
полным доступом к рабочей машине. Теория и настройка расписаны в
`docs/agent-security.md` — здесь остаётся довести до реально работающего состояния.

## Готово, когда

- [ ] Агент запускается с токеном `mn-agent`, а не с личным `gh auth login`
- [ ] У агента нет доступа к `~/.ssh` и `~/.config/gh`
- [ ] Любой клон репы поднимается одной командой

## Задачи

### P01-T1 — Урезанный токен 🔜
Пошаговая инструкция готова в `docs/agent-security.md`, осталось выполнить.

- [ ] Создать fine-grained токен `mn-agent` (Administration = No access)
- [ ] Проверить урезанность: `DELETE .../branches/main/protection` должен вернуть 403
- [ ] Прописать `GH_TOKEN` в окружение агента, убрать личный `gh auth`

### P01-T2 — Песочница 🔜
Ликбез по теме — `docs/learning-agenda.md` п.2.
