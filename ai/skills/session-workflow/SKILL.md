# Рабочая сессия по MN

Процесс описан в `docs/process.md`: Цели → Проекты → Задачи → Минизадачи.
Этот скилл — как агенту его вести.

## Когда использовать

- Начало сессии: «что делаем», «с чего начать», «покажи повестку».
- Конец сессии: «заканчиваем», «подведи итоги», «зафиксируй».
- Появилась новая идея или задача — куда её положить.
- Надо завести новый проект или цель.

## Открытие сессии

1. Прочитать `docs/next-session-agenda.md` — там повестка и контекст прошлого раза.
2. Показать пункты владельцу и по каждому спросить: **делаю** / **в бэклог** / **удалить**.
3. Открыть файл проекта, к которому относится выбранный пункт (`docs/projects/PXX-*.md`),
   прочитать задачу и её минизадачи.
4. Если минизадача явно больше одной сессии — предложить разбить её мельче,
   прежде чем начинать.

Не начинать работу, не сверившись с повесткой: в ней контекст, которого нет в коде.

## Во время работы

- Отмечать `[x]` по мере закрытия минизадач — сразу, не в конце.
- Новая мысль, не относящаяся к текущей задаче → в `docs/backlog.md`, не отвлекаться.
- Значимое решение → строка в `ai/context/decisions-log.md`; если развёрнутое —
  файл в `docs/adr/`.

## Закрытие сессии

Обязательная последовательность:

1. Проставить `[x]` в файле проекта, обновить поле **Обновлён**.
2. Обновить статус проекта, если сменился (и строку в `docs/projects/README.md`).
3. **Переписать `docs/next-session-agenda.md`:**
   - 1–3 пункта в «Взять в работу», с оценкой и одной строкой «зачем»;
   - обновить блок «Контекст прошлой сессии» — что сделали, открытые вопросы;
   - проставить дату.
4. Если что-то решили не делать — перенести в `docs/backlog.md`, а не удалять молча.
5. Коммит + PR (в `main` напрямую нельзя, см. `AGENTS.md`).

Шаг 3 — самый важный. Без него следующая сессия начнётся с восстановления контекста.

## Куда что класть

| Что | Куда |
|---|---|
| Идея без разбора | `docs/backlog.md` |
| Работа на недели | новый файл в `docs/projects/` из `_template.md` |
| Работа на день–неделю | задача `PXX-TN` внутри файла проекта |
| Работа на одну сессию | чекбокс-минизадача внутри задачи |
| «Зачем это всё» | `docs/goals.md` |
| Принятое решение | `ai/context/decisions-log.md` или `docs/adr/` |
| Факт о продукте | `ai/context/overview.md` |

## Правила, которые легко нарушить

- Не больше **1–2 проектов** в статусе 🚧.
- Новый проект — только привязанный к цели из `docs/goals.md`.
- Минизадача формулируется как результат, а не как «поработать над X».
- Не детализировать минизадачами то, что дальше ближайших задач.
